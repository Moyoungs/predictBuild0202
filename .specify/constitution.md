# Constitution: 교량 수량집계표 자동화 시스템

> SpecKit 프로젝트 헌법. 모든 후속 spec/plan/task는 이 문서의 원칙을 위배할 수 없다.

## 프로젝트 정체성

**이름(가칭):** quantity-aggregator
**목적:** 여러 개의 교량 수량산출서(Excel)를 입력받아 표준화된 총괄수량집계표(Excel)를 자동 생성한다.
**도메인:** 토목/건축 시공 수량 적산
**첫 번째 참조 프로젝트:** 갈전7교 (2026년 프로젝트 실제 데이터 기반)

## 핵심 원칙 (Core Principles)

### P1. AI의 역할은 "판단"만, "계산"은 코드가 한다
수량 합산, 할증률 적용, 소계/합계 산출은 **결정론적 코드**가 수행한다.
AI(Claude API)는 다음 4가지 판단에만 사용한다:
- 항목명 매핑 (사전에 없는 신규 공종명)
- 단위 정규화 (사전에 없는 신규 단위 표기)
- 집계 행/데이터 행 구분이 휴리스틱으로 실패한 경우
- 모호한 카테고리 분류

AI가 숫자를 "직접 합산"하는 일은 없어야 한다.

### P2. Rule-first, AI-fallback
처리 우선순위는 항상 다음과 같다:
1. **규칙 기반 파싱** (openpyxl + 휴리스틱): 80% 이상 처리 목표
2. **용어 사전 매핑** (YAML): 15% 처리 목표
3. **AI 판단** (Claude API): 나머지 5% 이하

1회 집계 생성에 AI 호출이 **30회를 초과하면 설계 결함**으로 본다.

### P3. 사전(Dictionary)은 누적 자산이다
AI가 판단한 매핑 결과는 사람이 검토 후 **용어 사전(YAML)에 저장**된다.
프로젝트가 쌓일수록:
- AI 호출 횟수는 줄고
- 정확도는 올라가며
- 비용은 절감된다.

### P4. 원본 집계 행은 데이터가 아니다
입력 엑셀의 "소계/합계/TOTAL/계" 행은 파싱 시 **반드시 제외**한다.
합계는 항상 코드가 원본 데이터로부터 재계산한다.
재계산값과 원본 합계가 다르면 **검증 리포트에 명시**한다.

### P5. 모든 AI 판단은 검토 가능해야 한다
AI 매핑 결과는 자동 반영하지 않는다.
- 사용자 UI(Streamlit)에서 **제안 형태로 제시**
- 사용자가 승인/수정/거부 가능
- 승인된 매핑만 최종 결과에 반영
- 거부/수정 내역은 사전 개선에 피드백

### P6. 출력 서식은 템플릿 복제
사내 총괄집계표 서식은 코드로 재현하지 않는다.
- 기존 00번 파일(또는 사내 표준 템플릿)을 **템플릿으로 복사**
- 수치만 해당 셀 위치에 **값으로 채움**
- 서식(글꼴, 테두리, 병합, 수식)은 원본 유지

### P7. 프로젝트별 차이는 설정 파일로 흡수
구조물 컬럼 구성, 할증률, 기타 프로젝트 고유 규칙은
코드 수정 없이 `project_config.yaml`로 관리한다.

### P8. 검증 리포트는 필수 산출물이다
결과 Excel에는 항상 "검증리포트" 시트가 포함된다:
- 원본 입력에 있던 소계/합계 값 vs 자동 재계산 값
- 매칭 실패 항목 목록 (AI 판단 이력 포함)
- 누락 가능성 경고

### P9. 한국 표준 표기를 내부 표준으로 쓴다
- 단위: `m3`, `m2`, `m`, `ton`, `kg`, `EA`, `본`, `개소`, `회`, `hr`, `포대`, `식`
- 콘크리트 규격: `{최대골재}-{설계기준강도}-{슬럼프(3자리)}` (예: `25-35-150`)
- 철근 규격: `H{지름}` 또는 `D{지름}` (예: `H25`, `D13`)

입출력에서 어떤 표기가 들어와도 내부에서는 위 표준으로 변환 후 처리.

### P10. MVP는 "작동하는 3개 시트"부터 시작한다
전체 14개 시트를 한 번에 만들지 않는다.
- **MVP v1**: 구조물공총괄 + 토공총괄 + 철근가공조립
- **MVP v2**: + 주요자재, 콘크리트자재집계
- **MVP v3**: + 골재, 시멘트, 이월수량 등 나머지

### P11. 비개발자 손에 도달해야 가치가 있다
이 시스템의 최종 사용자는 **Python을 모르는 토목 엔지니어**다.
- 배포 형태는 **단일 폴더 zip + .exe** (Phase 6)
- Python·pip·가상환경·터미널 명령어를 일절 요구하지 않는다
- 사전 업데이트는 `data/terminology.yaml` 텍스트 편집으로 가능해야 한다 (재빌드 금지)
- API 키는 exe에 하드코딩하지 않는다 (보안 + 키 교체 자유도)

## 기술 스택 제약

### 언어·프레임워크
- **Python 3.11+** (타입힌트 적극 활용)
- **Streamlit** (웹 UI; Flask/FastAPI 대신 선택)
- **openpyxl** (xlsx 읽기/쓰기), **xlrd** (xls 읽기), **pandas** (데이터 처리)

### AI
- **Claude API** (Anthropic SDK) 사용
- 모델: `claude-haiku-4-5-20251001`(단순 매핑), `claude-sonnet-4-6`(복합 판단)
- **Tool use(Structured Output)**로 JSON 응답 강제
- **Prompt caching** 적극 활용 (용어 사전 = 캐시 대상)
- API 키는 `.env`에만 존재, 코드에 하드코딩 금지

### 패키지 관리
- **uv** 사용 권장 (pip보다 10배 빠른 설치)
- `pyproject.toml` 단일 소스
- 잠금파일 `uv.lock` 커밋

### 테스트
- **pytest** + 갈전7교 실제 데이터를 픽스처로 사용
- 회귀 테스트: `예상_결과.json` 고정 후 수치 일치 검증
- AI 호출은 **mock**으로 테스트 (실제 호출은 별도 integration test)

## 경계(Non-Goals)

다음은 이 프로젝트가 다루지 **않는다**:
1. 산출근거(산출서) 시트의 원시 수량 재계산 (이미 집계된 값만 사용)
2. 도면 분석 / CAD 데이터 처리
3. 단가/공사비 산출 (수량 집계까지만)
4. MIDAS, AutoCAD 등 엔지니어링 SW 연동
5. 다국어 지원 (한국어 전용)
6. 다중 프로젝트 동시 처리 (1회 1프로젝트)

## 품질 기준

- **공종 매칭률**: 갈전7교 기준 100% 달성 (신규 프로젝트 투입 시 95% 이상)
- **수치 정확도**: 원본 00번 기준 검증 케이스에서 오차 0.01 미만
- **처리 시간**: 5개 파일(총 3MB 내외) → 10초 이내
- **AI 호출 비용**: 1회 집계당 평균 500원 이하 (Haiku 기준)
